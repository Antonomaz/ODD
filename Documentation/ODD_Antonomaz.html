The corpus The project ANTONOMAZ, in collaboration with the Bibliothèque Mazarine, aims to encode the Mazarinades's corpus in order to integrate it to a database usable by different audiences (researchers, students...) and to allow it to be studied. The Mazarinades are a serie of textual sources of different genres (pamphlets, poems, letters, etc.) published during the Fronde (1648 - 1653) mostly against the Cardinal Jules Mazarin. The project uses the works of Célestin Moreau (Bibliographie des Mazarinades), Emile Socard, Ernest Labadie and of Hubert Carrier (Presses de la Fronde). In addition, the project wishes to find sources not registered by any of these bibliographers to get a corpus as exhaustive as possible. We encoded fac-similes from these digital libraries: Gallica (Bibliothèque Nationale de France) Mazarinum (Bibliothèque Mazarine) Google Books Document identifiers Identifying the document One of our priorities is to clearly identify the text once encoded. We chose to give to each document an unique identifier consisting of the name of the digital library where the document was founded and its digital permanent identifier. So, for a gallica's one, it will beMoreau1234_GALL. The filename will match with the xml:id. This is going to be modified in the future. Here is the list for the different possibilities in naming documents: mazarinum: Moreau1234_MAZ gallica: Moreau1234_GALL Google Books: Moreau1234_GALL If the text comes from Carrier, Labadie or Socard, replace "Moreau" by the right name. If your text is registered in one of Moreau's supplements, the identifier will be: Moreau1suppl12_GALL. You can translate it like this: Moreau, first supplement, 12th in the list, from gallica. If your document is inside a collection, the identifier must be: "Moreau115pp.31-38_GALL". In addition, each document will get a type inside the TEI declaration which corresponds to the corpus it belongs to. For example, if the text is a burlesque mazarinade, write "Mazarinade_burlesque". The fileDesc This part of the header contains four parts : the titleStmt the respStmt the publicationStmt the sourceDesc The titleStmt This part is essential for the presentation of the encoded document. It has to contain two titles : a main title (source's title) and a sub title (ANTONOMAZ édition). The main title has a source attribute filled by default by "Moreau". If it's possible to get the Mazarinum's title, take it and replace "Moreau" by "BM". The same operation will have to be done for the title the bibl
<titleStmt>        <title type="main">Visions astrologiques de Michel Nostradamus sur                toutes les affaires de ce temps, et la confusion de Mazarin. En                vers burlesques</title>        <title type="sub">ANTONOMAZ édition</title> </titleStmt>
The respStmt This part contains the list of persons having worked on the document's encoding. It is necessary to indicate a persName with a role attribute and an affiliation and the ORCID number (if there is one) for each person, encoded inside a ref. If the person gets an ORCID number, don't indicate the affiliation since the information is already contained inside the ORCID's database.
<persName role="Chef_projet">        <forename>Karine</forename>        <surname>Abiven</surname>        <idno>0000-0001-9518-1040</idno> </persName>
The publicationStmt This part contains always the publisher, a date for the file's creation and eventually its correction, as well as the availibility which contains the licence. In our project, this part is always the same.
<publicationStmt>        <publisher>Projet ANTONOMAZ / Sorbonne Université</publisher>        <ref target="https://github.com/Antonomaz"/>        <date type="file_creation"         when="2026-04-20">26 avril 2021</date>        <availability status="restricted"         n="cc-by">         <licence target="https://creativecommons.org/licenses/by/4.0"/>        </availability> </publicationStmt>
The sourceDesc This part contains one element: the bibl. It gets the following elements: ref author persName title pubPlace publisher date extent note ref About the first element ref: in the attribute target, copy only the link to the fac-simile. Specifications about the element author If the author is not certain, add: @type:"alleged_author" @source: "Name of the attributor (a person, an institution...)" If the author is a faked one, add: a @type:"faked_author" About the element date: if someone dated the document -the date was not inscribed on the original document - give the identity of the attributor like this: @source:"Name of the attributor" Each author will get an isni's number (if there is one), encoded in a ref attribute. About the element note: there is three type: "format" : the document's format. "Bibliographer's name_identifier": each document has an unique number. If we say "3600", it means it's the 3600th document in the Bibliographie. If the bibliographer is Moreau, write "Moreau_identifier". "BM_identifier" : identifier from mazarinum (the Bibliothèque Mazarine's digital library) About the element persName: it is meant for recipients and dedicatees only. It gets two attributes: role which is filled by "received". ref which is filled by an isni's number. If your author or recipient is a noble, do not write its titles. The information is already contained inside the isni's database.
<persName role="received"        ref="isni:0000000121336441">        <forename>Gaston</forename>        <surname>France (De)</surname> </persName>
If your author is anonymus, remove persName element and just fill author with "Anonyme". If you have multiple authors, duplicate the author element as much as needed. If you have no placeName, remove the ref and source attributes from the element and write "Sans Lieu". In some cases, the placeName inscribed on the document is fake. So, it is necessary to indicate the false placeName in a first pubPlace element and then the true one in another placeName element. If you have no publisher, remove the ref and source attributes from the element and write "Sans Nom". About the second ref element: it contains a type filled with "BM_notice". In the target, copy only the link to the Mazarine's notice.
<sourceDesc>        <bibl>         <ref target="https://gallica.bnf.fr/ark:/12148/bpt6k118539m"/>         <title source="BM">Visions astrologiques de Michel Nostradamus                  sur toutes les affaires de ce temps. The msDesc Even if the Mazarinades are, for the most part, printed documents, the msDesc was useful to us. To ensure a good encoding, some elements are recommanded : msIdentifier which contains information used to properly identify the manuscript
<msIdentifier>           <settlement>Paris</settlement>           <institution>Bibliothèque Nationale de                      France</institution>           <repository>Réserve des livres rares</repository>           <idno type="cote">YE-4819</idno> </msIdentifier>
history which contains information about the history of the manuscript. We used it to report the presence or absence of a stamp with a boolean (true or false). True is valid for any stamp, the ones from the library keeping the physical copy included.
<history>           <provenance>            <stamp>False</stamp>           </provenance> </history>

Concerning the abstract, this element serves to indicate notes from different libraries about the document or from the Antonomaz team. This part is essential since it can give precious information about the text to the audience. We have different abbreviations for each library or projects: BM: Bibliothèque Mazarine BML: Bibliothèque municipale de Lyon BnF: Bibliothèque nationale de France BL: The British Library BnA: Bibliothèque nationale d'Autriche BEB: Bibliothèque de l'Etat de Bavière BncF: Bibliothèque nationale centrale de Florence Antonomaz: Antonomaz project. You can add libraries to the list, but do not forget to indicate the abbreviation here so everyone can follow your lead. Text encoding We have chosen to divide our encoding into steps. The first one is to get the cleanest metadata. Consequently, we have decided for a very simple first encoding of the text's body. In our corpus, however, some documents are different from the others, with a richer text's body's encoding: this is the result of our work before taking the decision to make metadata our main priority for the moment. So, we will describe the two states of encoding. Simple encoding First, at all levels our edition must contain a text element with the following namespace : @xmlns="http://www.tei-c.org/ns/1.0". It checks the validation of the TEI schema. Then, we use the element bodywith: p for prose parts.

lb for line beginning. Richer encoding We use the element body with: head: contains the elements pb for page beginning with a link pointing towards the right page on the fac-simile, title for the title on the front page, ref which contains a link to the printer mark and note for edition's informations. pb for page beginning. fw : has three types : "pageNum" for page numbers, "catch" for catches", "sig" for signatures. p for prose parts. l for versified parts lg for a stanza lb for line beginning. It can get two attributes : rend and break used for cut words. hi is used for any graphic distinction sic reports an error in the original text. corr indicates a correction made by the editor of the encoded text. foreign is used to indicate a word or a word group in a language different from the original one. It gets one attribute xml:lang note to encode any note in the original text. Its location on the page should be indicated with a place.
